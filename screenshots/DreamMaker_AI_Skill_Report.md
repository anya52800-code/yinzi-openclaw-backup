## 核心能力详解

### 1. 模型智能选择

| 场景 | 推荐模型 | 原因 |
|------|---------|------|
| 概念探索/快速迭代 | 即梦图片 4.5 | 速度快，3 分钟出图 |

| 批量图标生成 | 即梦 4.5 + 1K | 快速筛选，精选后 4K 重绘 |

### 2. 提示词工程标准化

**结构化 Prompt 模板**：
```
基于参考图的[风格描述]

主题：[游戏/活动主题]

元素：
- [元素 1]
- [元素 2]
- [元素 3]

配色：[具体色值描述]

氛围：[情绪/氛围词]

风格：[美术风格] + [技术规格]
```

**示例（暗黑不朽头像框）**：
```
基于参考图的哥特边框风格，红金配色+骨骼元素

主题：暗黑破坏神 - 安达莉尔+督瑞尔双魔王

元素：
- 铁链：锈蚀铁链缠绕边框
- 荆棘：黑色毒液滴落的荆棘
- 安达莉尔：绿色毒雾、蜘蛛肢体
- 督瑞尔：甲虫外壳、倒钩

配色：深红、毒绿、骨白、纯黑

氛围：压抑痛苦、双重折磨

风格：暗黑奇幻哥特、暗黑不朽UI、透明背景
```

### 3. 浏览器自动化实现

**技术方案**：
- 通过 JavaScript 注入操作 iframe 工作区
- 自动定位输入框、按钮、下拉菜单
- 异常处理：卡顿检测、自动刷新、错误重试

**关键代码逻辑**：
```javascript
// 查找 iframe 中的输入框
const iframe = document.querySelectorAll('iframe')[0];
const iframeDoc = iframe.contentDocument;

// 填入提示词
const input = iframeDoc.querySelector('textarea');
input.focus();
input.value = '提示词内容';

// 点击生成按钮
const buttons = iframeDoc.querySelectorAll('button');
buttons.find(b => b.textContent.includes('立即生成')).click();
```

## 问题与限制

| 问题 | 根因 | 解决方案 |
|------|------|---------|
| 生成进度卡在 0% | 服务器繁忙/账号异常 | 自动刷新页面或切换模型重试 |
| 元素定位失败 | DreamMaker 界面改版 | 更新 selector 或使用模糊匹配 |
| 参考图上传失败 | 路径权限/跨域限制 | 确保图片在允许访问目录 |
| 4K 生成时间长 | 服务器排队 | 异步执行，后台等待，完成后通知 |
| IP 敏感内容 | AI 模型训练数据限制 | 人工审核层，阻断风险内容 |
